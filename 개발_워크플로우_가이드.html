Step 1: 코드 수정 (HMI_REAL에서)

📁 수정 위치
• Frontend: HMI_REAL/frontend/src/
• Backend: HMI_REAL/backend/*.py
• Simulator: HMI_REAL/simulator/*.py

Frontend 수정 예시:

// HMI_REAL/frontend/src/components/FanDiagram.jsx // 팬 다이어그램 컴포넌트 수정 const FanDiagram = ({ sensors, fans, equipment, onCommand }) => { // 코드 수정... return ( <div className="fan-diagram-wrapper"> {/* UI 수정 */} </div> ) }

Backend 수정 예시:

# HMI_REAL/backend/main.py @app.post("/api/equipment/command") async def equipment_command(request: EquipmentCommand): # 새 API 엔드포인트 추가 result = await modbus_client.send_command(...) return {"success": True}

Step 2: 개발 환경에서 테스트

HMI_REAL 폴더에서: START_HMI.bat 더블클릭

• Frontend: Vite Dev Server (Hot Reload)
• 코드 저장 시 자동 새로고침
• 즉시 결과 확인 가능
• 빠른 반복 개발 가능

접속 정보
• Frontend: http://localhost:5173 (개발 서버)
• Backend: http://localhost:8000
• PLC Simulator: 192.168.0.130:502

Step 3: 배포 패키지 자동 생성 자동화

HMI_REAL 폴더에서: 배포_패키지_생성.bat 더블클릭
✅ 자동으로 수행되는 작업

[1/5] Frontend 빌드
• npm run build 자동 실행
• React 코드를 최적화된 정적 파일로 변환
• dist/ 폴더에 결과물 생성

[2/5] 배포 폴더 초기화
• HMI_DEPLOY 폴더의 기존 파일 삭제
• 깨끗한 상태로 초기화

[3/5] Frontend 파일 복사
• frontend/dist/ → HMI_DEPLOY/frontend/dist/
• 약 325KB 복사

[4/5] Backend 파일 복사
• Python 소스 코드 복사
• venv/ 폴더 복사 (약 37MB, 1-2분 소요)

[5/5] 기타 파일 복사
• Simulator 복사
• 실행 파일 (.bat) 복사
• 문서 파일 복사
• 배포_설명서.txt 자동 생성

예상 소요 시간: 2-3분

🐛 7. 문제 해결

Q1: 배포 패키지 생성 시 오류 발생

증상: 배포_패키지_생성.bat 실행 시 오류

해결 방법:
1. Frontend 빌드 오류 확인
   → cd frontend && npm run build 수동 실행
   → 오류 메시지 확인

2. 파일 권한 확인
   → HMI_DEPLOY 폴더가 읽기 전용인지 확인

3. HMI_DEPLOY 폴더 사용 중 확인
   → 다른 프로그램에서 파일 사용 중인지 확인
   → STOP_HMI.bat 실행 후 재시도

Q2: 배포 후 화면이 안 나옴

증상: HMI_DEPLOY 실행 시 빈 화면

해결 방법:
1. Frontend 빌드 최신 버전 확인
   → frontend/dist/assets/ 폴더의 파일 날짜 확인

2. 브라우저 캐시 삭제
   → Ctrl+F5 (강력 새로고침)
   → 브라우저 완전 종료 후 재실행

3. dist 폴더 확인
   → HMI_DEPLOY/frontend/dist/index.html 존재 확인
   → assets 폴더 내 파일 확인

Q3: 의존성 오류 발생

증상: ModuleNotFoundError, ImportError 등

해결 방법:
1. requirements.txt 최신 버전 확인
   → pip freeze > requirements.txt

2. venv 폴더 제대로 복사되었는지 확인
   → HMI_DEPLOY/backend/venv/ 폴더 확인

3. 필요시 venv 재생성
   → backend 폴더에서 python -m venv venv
   → venv\Scripts\activate
   → pip install -r requirements.txt

Q4: E/R 환기 화면이 빈 화면

증상: E/R 환기 탭 클릭 시 빈 화면

해결 방법:
1. SVG 파일 확인
   → frontend/src/assets/fan_diagram.svg 존재 확인

2. Frontend 재빌드
   → 배포_패키지_생성.bat 재실행

3. 브라우저 콘솔 확인
   → F12 → Console 탭에서 오류 확인
